# GroovyWeb Hackathon Checklist: Team Guidelines

Use this checklist to ensure all rules are followed before Demo Day.

## 🏁 Prep Phase
- [x] Form team (max 3 members) and pick team lead.
- [x] Submit idea via Google Form (Deadline: March 4).
- [x] Identify a real client pain point (Protecting client identity).
- [x] Document the "Before" manual process (`before_demo_manual_process.md`).
- [x] Setup tools (Codex, Antigravity, Opencode, Kilo Code).
- [x] Plan agent workflow on paper (Done via `orchestrator.md`).
- [x] Decide roles (Who codes, who documents, who presents).

## 🛠️ Build Phase
- [x] Build an AUTONOMOUS agent (No simple chatbots allowed).
- [x] Write agent architecture guidelines (Found in `agents/` and `context/`).
- [x] Define KRAs and KPIs (`KRA_KPIs.md`).
- [x] Capture photos of whiteboard planning.
- [x] Capture 15-30 sec video clips of coding in action.
- [x] Create LinkedIn/Instagram post and publish with @groovyweb and #AIAgentHackathon.

## 🎤 Demo Day
- [ ] Ensure Pitch doesn't exceed 3 minutes.
- [ ] Rehearse Live Demo (max 2 mins).
- [ ] Submit Code (GitHub Repo / Shared Link).
- [ ] Submit Documentation (This checklist, KRAs, System Architecture).
- [ ] Pitch Script opens with the Client Problem.
- [ ] Pitch Script ends with an Impact Statement.
- [ ] Prepare for Q&A with judges.

## 🚨 Final Rule Check
- [ ] Did we solve a real client problem? YES.
- [ ] Is it an autonomous agent? YES.
- [ ] Did we accidentally reveal the client name? NO.
- [ ] Is documentation complete? (Worth 20% of score!) YES.
